# 2.1 GPU 执行层次

## 要说明的问题

GPU 不是“很多线程随便同时跑”，而是有明确的层次组织方式。
理解这个层次，才能理解 TileLang 为什么要显式表达并行。

## 要点

- `Thread` 是最小执行单元
- `Warp` 是 SIMT 调度的基本单位
- `Block` 允许线程共享片上资源
- `Grid` 是整个 kernel 的启动范围
- 同一层级内的协同成本通常更低，跨层级通信成本通常更高
- 这也是为什么 tile 和同步边界必须一起设计

## 这里应该解释的现象

- 为什么 warp divergence 会拖慢性能
- 为什么 block 之间无法直接同步
- 为什么并行数量不等于有效吞吐
- 为什么“线程更多”不一定“更快”
