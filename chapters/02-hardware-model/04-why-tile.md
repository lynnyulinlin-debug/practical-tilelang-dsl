# 2.4 为什么需要 Tile

## 要说明的问题

Tile 不是随便切块，而是为了把计算、数据和硬件层次对齐。

## 要点

- 通过 tile 提高数据复用
- 通过 tile 把热点数据搬到更快的内存层
- 通过 tile 控制并行粒度
- 通过 tile 让编译器更容易生成高效代码
- 通过 tile 把“访问模式”改写成更适合硬件的局部模式
- 通过 tile 把大问题拆成可验证、可调优的小问题

## 这里应该解释的现象

- 为什么大多数高性能 kernel 都离不开 tile
- 为什么 tile 大小不是越大越好
- 为什么 tile 需要结合寄存器、共享内存和 block 粒度一起设计
- 为什么 tile 是性能、正确性和可维护性的共同折中点
