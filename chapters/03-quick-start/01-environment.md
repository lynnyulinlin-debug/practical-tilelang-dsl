# 3.1 环境准备

## 要说明的问题

教程不是在抽象环境里运行的。  
如果环境不明确，读者会在还没理解 kernel 之前就卡住。

## 要点

- 需要什么基础依赖
- 需要什么 Python 环境
- 需要什么后端支持
- 没有目标硬件时怎么做回退
- 先确认“能不能跑”，再考虑“跑得快不快”

## 当前仓库的实际口径

- 推荐使用单独的 `tilelang-tvm` conda 环境
- 只验证 CPU 路径时，`pip install apache-tvm` 已足够
- 需要 CUDA / GPU 路径时，再切换到支持 CUDA 的 TVM 包
- 当前机器若没有可用 NVIDIA 驱动，GPU 示例会被跳过
- 本章的环境目标不是“统一所有机器”，而是给出清晰的最小可执行路径

## 这里应该配的占位

**表：环境依赖与最低要求**

> 这里要说明 Python、TVM、目标后端、驱动版本等依赖的关系，并区分“必须项”和“可选项”。
> 对当前仓库，至少要说明：
> - CPU 验证：Python + `apache-tvm`
> - GPU 验证：Python + CUDA 版 TVM + 可用 NVIDIA 驱动
> - 没有驱动时：只跑 CPU fallback，不阻塞文档阅读

## 这里应该解释的现象

- 为什么同一份代码在不同环境下行为不同
- 为什么先检查安装，再看代码
- 为什么教程必须给出 CPU fallback
- 为什么“环境不同”经常会被误判成“代码有问题”

## 这一节应该留下的判断标准

- 如果 `import tvm` 都过不了，先不要看 kernel 代码
- 如果只有 CPU 环境，就先走 CPU fallback，把接口和验证跑通
- 如果目标是 GPU，再去确认驱动、CUDA 包和后端支持

## 常见误区

- 误区 1：环境配置只是安装步骤
- 误区 2：同一份教程必须要求所有机器完全一致
- 误区 3：没有 GPU 就无法学习这一章

实际上，好的教学路径应该允许读者先在 CPU 上建立心智模型，再逐步过渡到 GPU。
