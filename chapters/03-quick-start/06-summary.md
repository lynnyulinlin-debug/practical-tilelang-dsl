# 3.6 本章总结

## 你应该记住什么

1. 先把环境跑通，再看更深的优化
2. 第一个 kernel 的作用是建立最小闭环
3. 编译流程是理解 TileLang 的关键入口
4. 正确性验证和排错是写 kernel 的固定步骤

## 你还应该能判断什么

- 一个问题是环境问题、编译问题还是实现问题
- 正确性验证和性能验证应该在什么顺序下做
- CPU fallback 为什么不是“次等方案”，而是重要的学习路径
- 哪些错误应该先缩小输入规模再查
