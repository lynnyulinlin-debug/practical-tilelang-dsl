# 第 4 章导言：分块、内存层次与数据复用

## 章节定位

这一章是从“会写 kernel”走向“写得快”的第一道门槛。  
核心任务是把 TileLang 的内存原语和硬件层次对应起来，并解释为什么 tile 是性能设计的基础单位。

## 本章目标

- 建立 `global / shared / fragment` 的对应关系
- 解释 coalescing、alignment、bank conflict
- 说明数据复用如何改变算术强度
- 为后续调度优化和实战案例打底
- 让读者形成一个新的判断单位：tile 不是“切一刀”，而是“决定数据怎么流、在哪一层停留、被谁复用”的优化边界

## 本章判断标准

读完这一章，读者应该能判断：

- 哪些数据适合尽早放进更快的层次
- shared memory 什么时候是收益，什么时候是负担
- fragment / register 为什么更接近计算单元
- tile 的大小和形状为什么会影响性能和资源占用

## 本章最小深度要求

这一章至少要让读者说清楚：

1. 为什么 tile 和内存层次必须一起看
2. 为什么 shared memory 不是所有算子的默认答案
3. 为什么 fragment / register 更接近计算单元
4. 为什么 GEMM 是最好的解释案例

如果只能把 tile 理解成“切块”，这章还没有真正学会。
