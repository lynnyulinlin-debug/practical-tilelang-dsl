# 4.6 本章总结

## 分块、内存层次和数据复用怎么判断

1. tile 的本质是把计算和硬件层次对齐
2. shared memory 的核心价值是复用
3. fragment 和寄存器是更靠近计算单元的层次
4. GEMM 是理解这些概念的最好例子

这一章真正建立起来的，不只是几个内存层次的名字，而是“数据应该停在哪里、为什么停在那里、停多久最划算”的判断能力。  
只要这个判断成立，tile 就不再只是切块，而是性能设计的边界。

所以第 4 章真正建立的，是一个可以复用到后面所有章节的判断句：
数据不是只要搬进来就行，而是要决定它在每一层停多久、为什么停、什么时候离开。

哪种数据适合放在全局层，哪种更适合片上层，shared memory 的收益是否足以覆盖同步和占用成本，fragment / register 是否已经过度压缩了并发度，tile 的形状是否和数据流以及硬件层次一致，都是这一章要能说清楚的判断点。  
这一章已经不只是“介绍术语”，而是要开始拿这些术语做判断。  
如果只能背出 `global / shared / fragment`，但不会判断它们在具体算子中的作用，这章还不够。  
更进一步说，应该能解释：为什么同样是分块，有的分块在帮忙复用，有的分块只是在制造同步和占用成本。
