# 4.6 本章总结

## 你应该记住什么

1. tile 的本质是把计算和硬件层次对齐
2. shared memory 的核心价值是复用
3. fragment 和寄存器是更靠近计算单元的层次
4. GEMM 是理解这些概念的最好例子

## 你还应该能判断什么

- 哪种数据适合放在全局层，哪种更适合片上层
- shared memory 的收益是否足以覆盖同步和占用成本
- fragment / register 是否已经过度压缩了并发度
- tile 的形状是否和数据流以及硬件层次一致
