# 6.6 本章总结

## 你应该记住什么

1. Vector Add 用来建立最小闭环
2. Naive GEMM 用来暴露优化空间
3. shared memory GEMM 用来说明数据复用
4. fragment / Tensor Core GEMM 用来说明硬件匹配
5. 基础优化方法会在后续高级算子里继续复用
6. 如果要做教学展示，Notebook 适合逐格解释流程，`.py` 适合保留可测试实现

## 你还应该能判断什么

- 一个 GEMM 现在更像是正确性问题、复用问题还是硬件匹配问题
- 什么时候停在 shared memory 版本就足够
- 什么时候应该继续进入 fragment / Tensor Core 路径
- 后续调优应该优先看结构还是参数

## 进入下一章之前
