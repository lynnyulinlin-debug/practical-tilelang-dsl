# 6.6 本章总结

## 先记住什么

1. Vector Add 用来建立最小闭环
2. Naive GEMM 用来暴露优化空间
3. shared memory GEMM 用来说明数据复用
4. fragment / Tensor Core GEMM 用来说明硬件匹配
5. 基础优化方法会在后续高级算子里继续复用
6. 如果要做教学展示，Notebook 适合逐格解释流程，`.py` 适合保留可测试实现

这一章真正想留下的，不是三个 GEMM 版本本身，而是一条很稳定的优化顺序：先让算子跑通，再把重复访存压下去，最后才谈硬件匹配。  
只要这个顺序成立，后面的 attention、融合算子和跨平台迁移就都有了同一套判断基础。

## 这章最后要判断什么

- 一个 GEMM 现在更像是正确性问题、复用问题还是硬件匹配问题
- 什么时候停在 shared memory 版本就足够
- 什么时候应该继续进入 fragment / Tensor Core 路径
- 后续调优应该优先看结构还是参数

## 下一章怎么接上

下一章会把复杂度提上去，开始处理 FlashAttention 和融合注意力这类高级算子。

## 这一章的收口方式

这一章的任务不是把所有 GEMM 变体都写满，而是让读者通过几个版本建立优化顺序感。  
如果看完后还能明确说出“先做什么、再做什么、什么时候停”，这一章就算达到了目标。

换句话说，这一章的收口不是“学会了三个实现”，而是“学会了怎么判断一个算子该停在哪一层”。  
这比记住某一个代码片段更重要。
